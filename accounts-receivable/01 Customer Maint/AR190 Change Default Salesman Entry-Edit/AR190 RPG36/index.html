    Business Rules¶

    The program enforces the following business rules for managing salesman change transactions:

    1. Company Number Validation:
    2. The company number (CO) must not be blank/zero (MSG1: "COMPANY NUMBER CANNOT BE BLANK").

    3. The company number must exist in the ARCONT file (MSG2: "COMPANY NOT IN CONTROL FILE").

    4. Customer Number Validation:

    5. If a new salesman code (SLN,X) is entered, a customer number (CUS,X) must be provided (MSG3: "CUSTOMER CANNOT BE ZERO" and MSG4: "WHEN NEW SALESMAN IS KEYED").

    6. The customer number must exist in the ARCUST file (MSG5: "INVALID CUSTOMER NUMBER").

    7. Salesman Code Validation:

    8. The new salesman code (SLN,X) must exist in the GSTABL file under the SLSMAN table (MSG6: "INVALID NEW SALESMAN CODE").

    9. A previous rule requiring the new salesman to differ from the old salesman (SLN,X SLO,X) was removed on 09/27/17 (lines 02470264).

    10. Delete Code Validation:

    11. The delete code (DL,X) must be ' ' (blank), 'A', or 'D' (MSG7: "DELETE CODE MUST BE ' ',A, OR D").

    12. If DL,X = 'D', the record is marked for deletion, and if it doesnt exist in ARSLST, it is deleted via EXCPTDELREC.

    13. Transaction Processing:

    14. If a customer number is entered:
      • If the record exists in ARSLST and DL,X 'D', it is updated (UPDREC).
      • If the record does not exist and DL,X 'D', it is added (ADDREC).
      • If the record does not exist and DL,X = 'D', it is deleted (DELREC).

    15. Deleted records in ARSLST (ASDEL = 'D') are flagged with MSG8 ("THIS RECORD PREVIOUSLY DELETED").

    16. Screen Navigation:

    17. Screen S1 collects the company number and optional customer number.
    18. Screen S2 displays up to 10 transactions and supports roll-up/roll-down for navigation.
    19. If no records are entered or the end of the file is reached, the program returns to screen S1.
    20. The end of the file (ARSLST) triggers MSG9 ("END OF FILE HAS BEEN REACHED").

    21. The beginning of the file triggers MSG10 ("BEGINNING OF FILE HAS BEEN REACHED").

    22. Field Protection:

    23. Customer numbers for existing records are protected (indicators 5059) to prevent modification.

    24. Error indicators (4049, 6069, 7079) highlight invalid fields on screen S2.

    25. Data Retrieval:

    26. Customer names are retrieved from ARCUST (ARNAME).
    27. Salesman names are retrieved from GSTABL (TBDESC) for the SLSMAN table.
    28. The old salesman code is retrieved from ARCUST (ARSLMN) or ARSLST (ASSLSO).

    BB943V RPGLE

    The provided document is an RPGLE program named BB943V, called from the OCL procedure BB943INQ.ocl36.txt and referenced by the BB943 program in the IBM System/36 (or AS/400 compatibility mode) environment. This program, titled "Customer Sales Agreement File Maintenance Validation," is designed to validate input data for customer sales agreements before they are added or updated in the bicuag file by BB943. Below, Ill explain the process steps, business rules, tables used, and external programs called, based on the provided RPGLE source code, the OCL context, and its integration with BB943 and BB943P.

It receives a data structure (CUAGDS) from BB943, validates fields against multiple files, and returns error messages and cursor positioning information if validation fails.     Business Rules¶

    The program enforces the following business rules for validating customer sales agreement data:

    1. Field Presence and Format:

    5. PPD must be 'P' or blank (COM(55)).
    6. PRIM must be 'I' (invoice) or 'M' (month-end) (COM(29)).

    7. Either PRCE or OFFP must be non-zero (COM(14)).

    8. Key Field Validation:

    10. Customer (CUST): Must exist in ARCUST (COM(06)).
    11. Location (ALOC): Must exist in INLOC (COM(20)).
    12. Container (KCNTR): Must exist in GSCNTR1 (COM(33)). Blank for non-fluid products (COM(17)), required for fluid products (COM(54)).
    13. Container Type (TCCNTY): Must exist in GSTABL with CNTRTY prefix (COM(32)).
    14. Product Codes (S3PR): Must exist in GSPROD, no duplicates (COM(11), COM(51)), and at least one valid code (COM(09)).
    15. Unit of Measure (UNMS): Must exist in GSTABL (COM(08), COM(50)).
    16. Purchase Order (PORD): Must exist in BBORDH if S3POOR is 'P' or 'O' (COM(13)).

    17. Customer-Ship-to (S3CS): Validates customer (s3cscu) in ARCUST (COM(56)) and ship-to (s3cssh) in SHIPTO (COM(57)).

    18. Date and Time Validation:

    19. Start Date/Time (STDT, STTM): Must be valid, with hours 0023 (COM(21)) and minutes 0059 (COM(22)). Not all zeros (COM(25)).
    20. End Date/Time (ENDT, ENTM): Must be valid, with hours 0023 (COM(23)) and minutes 0059 (COM(24)). Not all zeros (COM(26)). Must follow start date/time (COM(35)).

    21. Password for Backdated Start: If STDT is more than 3 days ago, a valid password (KAGPW) is required (COM(37)).

    22. Duplicate Agreement Checks:

    23. For add operations (ADDREC = 'Y'), ensures no duplicate agreement exists in BICUA3 based on key fields (CONO, CUST, ALOC, KCNTR, UNMS, SHIP, S3PX, PORD, MNQY, MXQY, STDT, STTM, FRCD) (COM(58)).

    27. "Copy to other" processing (e.g., copying agreement to other locations, ship-tos, or customer-ship-tos) is allowed only when adding a record (ADDREC = 'Y', via F6 or option 3 copy).

    29. The S3EXPR (expire) field is only editable for option 3 (copy). If S3EXPR = 'Y', the original record is expired (COM(47)).

    30. Price and Quantity Fields:

    31. PRCE and OFFP are expanded to 9.4 packed decimal (per JK03).

    32. MNQY and MXQY are validated as part of the agreement key (per JK02).

    33. Program Termination:
